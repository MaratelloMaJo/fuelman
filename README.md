# ⛽ FuelMan — Умный учёт расхода топлива

**FuelMan** — это современное кроссплатформенное мобильное приложение для умного, детального учёта расхода топлива, электроэнергии и общих расходов на содержание вашего автомобиля (или целого автопарка). 

Приложение спроектировано с акцентом на локальность хранения, конфиденциальность, высокую точность расчётов и удобство ежедневного использования.

---

## 🛠️ Стек технологий (Tech Stack)

Проект разработан на базе современных инструментов и библиотек экосистемы Flutter:

*   **Фреймворк:** [Flutter](https://flutter.dev/) (SDK `>=3.5.0`) — создание адаптивного и плавного UI для Android и iOS.
*   **Язык программирования:** [Dart](https://dart.dev/) — строго типизированный язык с высокой производительностью.
*   **Управление состоянием и Навигация:** [GetX](https://pub.dev/packages/get) — производительный менеджер состояний, упрощающий инжектирование зависимостей (Dependency Injection) и реактивный роутинг.
*   **База данных:** [SQLite (sqflite)](https://pub.dev/packages/sqflite) + [path](https://pub.dev/packages/path) — надёжное локальное реляционное хранилище для мгновенного доступа к истории заправок и расходов.
*   **Построение графиков:** [fl_chart](https://pub.dev/packages/fl_chart) — интерактивные и красивые графики стоимости и расхода топлива/электроэнергии.
*   **Локализация:** [intl](https://pub.dev/packages/intl) + `flutter_localizations` — поддержка мультиязычности и форматирование дат/чисел под стандарты выбранного региона.
*   **Аппаратное API и службы:**
    *   [geolocator](https://pub.dev/packages/geolocator) — автоматическое определение координат заправочных станций по GPS.
    *   [share_plus](https://pub.dev/packages/share_plus) — экспорт данных в CSV и передача резервных копий БД.
    *   [shared_preferences](https://pub.dev/packages/shared_preferences) — сохранение пользовательских настроек (тема, валюта, единицы измерения).
    *   [flutter_local_notifications](https://pub.dev/packages/flutter_local_notifications) — напоминания о необходимости внести показания.
    *   [file_picker](https://pub.dev/packages/file_picker) — выбор файлов бэкапа для восстановления.
    *   [url_launcher](https://pub.dev/packages/url_launcher) — запуск внешних карт для построения маршрута к заправкам.

## 🌟 Ключевые преимущества (Key Advantages)

1.  **Универсальность типов двигателей:** Поддерживает не только стандартные бензин и дизель, но и все виды гибридов (PHEV, HEV, MHEV, BEV+REx) и чистые электромобили (BEV), позволяя вести учёт расхода в киловатт-часах (кВт⋅ч) и литрах/галлонах параллельно.
2.  **Высокоточный расчёт (Full-to-Full):** Автоматический расчёт расхода на основе метода «полного бака». Также поддерживаются частичные заправки с накопительным пересчётом расхода при следующей заправке «до полного».
3.  **Контроль аномалий и ошибок:** Приложение предупреждает о потенциальных ошибках ввода (например, если новый пробег на одометре меньше предыдущего, пройденное расстояние аномально мало или велико, или введён слишком большой объём). Аномальные записи можно сохранить, пометив флагом, чтобы они не искажали графики аналитики.
4.  **Полная приватность (Privacy First):** Все данные хранятся исключительно на вашем устройстве в локальной SQLite базе данных. Никаких внешних серверов, облачных трекеров или скрытой телеметрии.
5.  **Резервное копирование и Импорт:** Возможность экспортировать локальную базу данных SQLite в файл `.db` для резервной копии и легко восстанавливать данные при переносе на другое устройство.
6.  **Учёт расходов на уход и обслуживание:** Раздельный учёт заправок и сопутствующих трат, распределённых по категориям (Сервис/ТО, Мойка, Масло, Шины, Налоги/Страховка, Запчасти, Другое).
7.  **Локализация «из коробки»:** Полная поддержка трех языков — Русский (🇷🇺), English (🇬🇧) и Қазақша (🇰🇿) с возможностью независимого выбора валют (RUB, KZT, USD, EUR) и мер измерения (Литры, Галлоны).
8.  **GPS-трекинг заправок:** Сохранение точных координат места заправки/зарядки с возможностью открыть точку на картах Google Maps или Apple Maps в один клик.

## 🚀 Инструкция по установке и запуску

Для сборки приложения вам понадобится установленный [Flutter SDK](https://flutter.dev/docs/get-started/install) версии `>= 3.5.0`.

### 1. Клонирование репозитория
```bash

```

### 2. Установка зависимостей
Скачайте все пакеты, описанные в `pubspec.yaml`:

dart run flutter_native_splash:create

Запустите приложение на подключённом мобильном устройстве или эмуляторе:
```bash
flutter run
```

## 📈 Логика расчёта расхода и обработка аномалий

### Алгоритм расчёта расхода топлива (Full-to-Full)
*   **Полный бак (Full):** Когда пользователь отмечает заправку флагом «Полный бак», приложение ищет в истории предыдущую заправку «до полного» и рассчитывает средний расход топлива по формуле:
    
    $$\text{Расход} = \frac{\text{Суммарный объём заправленного топлива за период (л)}}{\text{Разница показаний одометра (км)}} \times 100$$
    
*   **Частичная заправка (Partial):** Если бак заправлен частично, объём топлива суммируется в буфер. Расчёт расхода откладывается до следующей полной заправки, чтобы учесть весь объём топлива и точную дистанцию, пройденную за несколько частичных заправок.

### Контроль аномалий ввода
Для защиты графиков статистики от искажений из-за опечаток, приложение валидирует каждую запись:
*   **Падение одометра:** Блокирует сохранение, если новое значение одометра меньше предыдущего.
*   **Малый пробег:** Предупреждает, если между заправками пройдено менее 10 км (расход топлива на такой дистанции не рассчитывается из-за высокой погрешности).
*   **Большой пробег:** Предупреждает при пробеге более 3000 км между записями.
*   **Аномальный объём:** Сверяется с указанным в карточке авто объёмом бака.
*   Если запись сохраняется как аномальная, она получает флаг `isAnomaly = 1` и полностью исключается из формул расчёта среднего расхода, сохраняя при этом общие затраты.

## 🔒 Безопасность и хранение данных

*   **Локальное хранение:** Локальная СУБД SQLite гарантирует высокую скорость работы и полную автономность без доступа к сети.
*   **Экспорт в CSV:** Удобная выгрузка истории заправок и расходов в универсальный табличный формат CSV для импорта в Excel или Google Таблицы.
*   **Бэкапы БД:** Экспортируйте файл базы данных SQLite `fuel_entries.db` в память телефона для надёжного резервного сохранения ваших данных.
